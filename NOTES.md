
# Aloxa — Project Constraints

## C1: Cost

| ID    | Constraint                                                                                                       |
| ----- | ---------------------------------------------------------------------------------------------------------------- |
| C1.1  | The total hardware cost of the project shall not exceed £100.                                                    |
| C1.2  | The project shall rely on free or open-source software wherever possible to avoid licensing costs.               |
| C1.3  | Any external service used (weather, health knowledge) shall be available within a free tier or trial allowance.  |
| C1.4  | All electronic components, enclosures, and 3D-printed parts shall be sourced within the £100 hardware budget.    |

## C2: Time

| ID    | Constraint                                                                                                                |
| ----- | ------------------------------------------------------------------------------------------------------------------------- |
| C2.1  | The project shall be completed within a one-year development window.                                                      |
| C2.2  | The project shall reach a working end-to-end prototype before the final third of the development window.                  |
| C2.3  | The remaining time shall be reserved for testing, refinement, and writing up the project.                                 |

## C3: Scope

| ID    | Constraint                                                                                                                       |
| ----- | -------------------------------------------------------------------------------------------------------------------------------- |
| C3.1  | The project shall target a single Raspberry Pi 5 device intended for one user in a home setting.                                 |
| C3.2  | The project shall support the English language only.                                                                             |
| C3.3  | The project shall provide medication reminders and information only; it shall not provide medical diagnosis, treatment, or prescription advice. |
| C3.4  | The project shall not integrate with any hospital, pharmacy, or clinical record system.                                          |
| C3.5  | The project shall not include user accounts, multi-user profiles, or remote access from another device.                          |
| C3.6  | The project shall not include a mobile or web application.                                                                       |
| C3.7  | The project shall include only the features listed in the functional requirements; additional features are out of scope.         |
| C3.8  | Hardware support shall be limited to the chosen microphone, speaker, indicator lights, and pill compartments; no other peripherals are in scope. |
